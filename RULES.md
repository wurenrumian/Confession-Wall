# 项目贡献规范

## 提交规则

### 1. 必须通过 PR 提交更改
- 所有代码更改必须通过 Pull Request (PR) 提交
- 禁止直接推送到主分支（main/master）

### 2. 不要提交不必要的文件
- 不要提交 IDE 配置文件（.vscode/, .idea/）
- 不要提交操作系统生成文件（.DS_Store, Thumbs.db）
- 不要提交依赖文件夹（node_modules/, vendor/）
- 不要提交日志文件（*.log）
- 不要提交临时文件（*.tmp, .temp/）
- 不要提交环境配置文件（.env, config.local.js 等包含敏感信息的文件）
- 不要提交编译输出文件（dist/, build/, .output/）

**不确定是否该提交的文件？可以询问 AI 助手或查看 .gitignore 文件。**

### 3. 问题反馈优先使用 Issue
- 遇到 bug 或有新功能建议，请先在 [Issues](../../issues) 中创建工单
- 提供详细的复现步骤、预期行为和实际行为
- 附上相关截图或错误信息

## 代码规范

- 遵循项目的代码风格（使用 ESLint/Prettier 配置）
- 提交前请确保代码通过测试
- 提交信息请使用清晰的中文或英文描述

## 分支管理

- 主分支：`main`（稳定版本）
- 开发分支：`dev`（开发中版本）
- 功能分支：`feature/xxx`（新功能）
- 修复分支：`fix/xxx`（bug 修复）

## 工作流程

1. 克隆本仓库到你的 GitHub 账号
2. 在本地新分支中进行开发和本地测试
3. 远程推送到仓库
4. 创建 PR 到本仓库的 `main` 分支
5. 等待审核和合并
